Responsive Web Design

This module is a very nitty-gritty thing to do. However, it is probably one of the most important! If people are looking at your webpage with different screen dimensions, you want it to fit properly. Whether they are on their phone, tablet, laptop, or desktop, shouldn't matter. Your webpage should fit and fix itself accordingly. As a student, I learned that many things happen behind the scenes. Before, when I went onto a website from a phone and desktop, I didn't even take a second to realise how different it was, and how much work was really there. I learned how to use the user's viewport in order to adjust the webpage to zoom, or size of display.

CSS Flexbox

Probably the most important part of making a webpage look organized. Moving around items, having containers inside containers, and floating them all over your page. The funny thing is, change 1 variable, and EVERYTHING collapses. One of the most painful proccesses I have encountered. However, this entire webpage uses the flexbox as it's main component. The buttons, the headers, even my profile name, all contain flex.

CSS Grid

The CSS grid is another way to organize items inside a container. My rubik's cubes are positioned in a grid layout. It is much simpler to use than flexbox, but not as free-flowing and smooth. It's a grid... "Modern" style websites with sleek sharp edges could benefit from the CSS grid. but more cartoonish and smoother web pages are much better without it. You can see a clear difference when it comes to CSS grid versus flexbox. The buttons use flexbox, the cubes use grid. Very different styles. It is more about how to use it, and when to know where to use it. Sometimes there is just a right time and a right place.
